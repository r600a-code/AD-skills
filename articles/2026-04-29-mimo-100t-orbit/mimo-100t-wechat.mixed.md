# 不是送福利，是在筛选谁能进入小米 MiMo 的轨道

![不是送福利，是在筛选谁能进入轨道](/Users/aiad/Desktop/claw/weixingzh/articles/2026-04-29-mimo-100t-orbit/assets/screenshot-20260430-105224.png)

很多人看到 MiMo 100T，第一反应是：小米开始发福利了。


我一开始也这么看。

但把活动页、MiMo 官网、开放平台文档页、新闻页一起翻完之后，我的判断变了。

这不是普通送券。

这更像一次公开的生态筛选。

![不是送福利，是在筛选谁能进入轨道](/Users/aiad/Desktop/claw/weixingzh/articles/2026-04-29-mimo-100t-orbit/assets/02-track.png)

先把事实摆出来。

MiMo Orbit 不是一个孤立活动。它是跟 MiMo-V2.5 的开源、公测、开放平台，一起推出的一套动作。开放平台新闻页写得很明确：小米 MiMo-V2.5 系列开源，Orbit 百万亿 Token 计划同步启动。也就是说，这不是模型发完以后顺手做点市场活动，而是模型、平台、激励、生态接入，一起往前推。

第二个事实，是额度真的很大，而且写得很死。活动页和开放平台新闻页都写了：30 天内，面向全球用户，发放总计 100 万亿 Token 权益，赠完即止。活动时间是北京时间 2026 年 4 月 28 日 00:00 到 5 月 28 日 00:00。

但真正值得看的，不是 100T 这三个字。

而是它怎么发。

这不是点一下就领的普惠活动。

这是申请制。人工评估制。分层发放制。

我实际打开申请页之后，流程很清楚：先提交表单，等待大约 3 个工作日评估，通过后收到邮件，权益会在邮件后的 24 小时内到账。它问的东西也很具体：你常用哪些 Agent 或 AI 开发工具，你主要用哪些底层模型，你到底做出了什么项目，有没有账单截图、工作流截图、GitHub 链接、在线 demo。

这个味道一下就不一样了。

它不是在找“对 AI 感兴趣的人”。

它是在找，已经在用 AI 干活的人。

![不是围观者，是已经在干活的人](/Users/aiad/Desktop/claw/weixingzh/articles/2026-04-29-mimo-100t-orbit/assets/03-builders.png)

再往下看，还有两层信息很关键。

第一层，是奖励不是统一礼包。

FAQ 里写得很直接：平台会根据你提交的内容和账号状态，发放不同额度、不同档位的 Token Plan，或者不同金额的赠金，以实际到账为准。开放平台新闻页又补了一层更具体的信息：申请通过的用户，最高可以拿到 Max 档位 Token Plan，包含 16 亿 Credits，标价 659 元。

这说明它不是在平均撒钱。

它是在分层配资源。

第二层，是它真正想抢的，不只是试用用户，而是工作流入口。

活动页里直接写，MiMo API 开放平台可以接 Claude Code、Cursor、OpenClaw 这些主流编程工具。我又去翻了开放平台文档页，发现它不是嘴上说接入。文档里已经单独放了 Claude Code、OpenClaw、Hermes Agent、Codex、Cline、Kilo Code、Qwen Code、Zed、Roo Code、OpenCode、Cherry Studio、TRAE 的配置说明。

这点很重要。

因为这说明 MiMo 想争的，不只是“你来网页上聊一会儿”。

它想争的是：你的 IDE，你的 Agent，你的默认调用链路。

![真正想抢的，是你的工作流入口](/Users/aiad/Desktop/claw/weixingzh/articles/2026-04-29-mimo-100t-orbit/assets/04-workflow.png)

再补一层。

Orbit 计划本身，其实不是只面向个人创作者。

开放平台新闻页把它拆成了两部分：一部分是面向 AI builder 的“创造者百万亿 Token 激励计划”；另一部分是面向 Agent 框架团队的“Agent 生态共建计划”。后者说得更直接：MiMo 会给 Agent 框架团队提供专项支持，甚至提供限免支持，让对方的用户零门槛接入 MiMo 系列模型。新闻页还点名提到了 OpenCode、Hermes Agent、KiloCode 这些合作对象。

这就不是单纯获客了。

这是在做生态插桩。

而这件事的上游，又不是封闭策略。

MiMo-V2.5 这轮动作，本身就是开着做的。新闻页写明，这次采用 MIT 协议，全量开源，支持商用推理部署和二次训练，无需额外授权。两款模型都支持 100 万上下文窗口：MiMo-V2.5-Pro 主打复杂任务、Agent 与 Coding；MiMo-V2.5 主打原生全模态，支持文本、图像、视频和音频理解。

你把这些东西连起来看，味道就很明显了。

不是模型先关着，流量先收着。

而是开源、开放平台、Token 激励、框架接入，四件事一起推。

![100T 只是表面，真正发出去的是轨道资格](/Users/aiad/Desktop/claw/weixingzh/articles/2026-04-29-mimo-100t-orbit/assets/05-orbit.png)

那这背后的价值观，到底是什么？

我觉得很清楚。

第一，不是所有用户都一样。

过去很多平台喜欢讲普惠。先把量做上去，先把注册堆起来，剩下的以后再说。但 MiMo 这次不是这个逻辑。它写得很直白：资源要交到真正用 AI 进行创造的用户手里。

这不是高冷。

这是现实。

因为今天最贵的，已经不是注册用户。

而是能把模型变成真实产出的那批人。

第二，平台竞争正在从“谁有模型”，转向“谁能组织起开发者”。

模型参数会被追平。价格会被打穿。榜单也会不断刷新。

但谁能先把开发者、工具链、工作流、框架团队，稳稳吸进来，谁才真正有生态护城河。

你会发现，MiMo 这次最狠的地方，不在 100T。

而在它把 Token、文档、开源、适配、招募，一次性串成了闭环。

这不是撒钱。

这是铺轨道。

第三，免费不是慈善。免费是筛选机制。

很多人看到免费 Token，第一反应是补贴。

但平台视角不是这么看的。

免费额度，尤其是这种带人工审核、带证明材料、带分层发放的免费额度，本质上是一种资源投资。投给谁，不投给谁，背后就是平台的价值判断。

说白了，真正的信号不是“发多少”。

而是“发给谁”。

第四，今天最值钱的，不是会聊天的用户，而是能把模型接进现实的人。

申请表为什么要看你的项目？为什么要看你的截图、账单、GitHub、demo？为什么强调 Agent、Coding、工作流？

因为平台已经不满足于“你觉得它好不好用”。

它要看的是：你能不能把这套能力接进真实世界，然后放大。

所以这件事最后最值得记住的一句话，不是 100 万亿。

而是：AI 平台开始越来越认真地区分，谁只是来围观，谁是真的在拿模型干活。

这可能才是 MiMo 100T 背后，真正的价值观。
