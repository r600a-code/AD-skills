# 设计原则 · Design Principles

本 skill 的设计原则融合了 **Vogue 审美**和 **Guizang-ppt 杂志风**。

---

## 核心哲学

### 1. 克制优于炫技

- 不用阴影、浮动卡片、渐变装饰
- 一切信息靠**字号对比 + 字体对比 + 留白**
- 视觉张力来自排版结构，不是装饰元素

### 2. 结构优于装饰

- 大号数字序号（01/02/03）制造视觉锚点
- 等宽标签（全大写、字母间距）提供元信息
- 左对齐分割线引导阅读动线
- 黑底引用框制造视觉停顿

### 3. 字体分工明确

| 字体类型 | 用途 | 视觉作用 |
|----------|------|----------|
| 衬线（Noto Serif SC） | 标题、金句 | 视觉重音、高级感 |
| 非衬线（Noto Sans SC） | 正文、描述 | 信息密度、可读性 |
| 等宽（SF Mono） | 标签、日期 | 装饰节奏、技术感 |

**核心规则**：
- 衬线 = 标题 = 视觉重音
- 非衬线 = 正文 = 信息密度
- 等宽 = 元数据 = 装饰节奏

### 4. 版面有攻击性（Vogue 核心）

**不要**：
- 居中排版（除结尾金句）
- 圆角卡片
- 渐变装饰
- 阴影
- 浮动元素

**要**：
- 左对齐
- 直角边框
- 强烈的字号对比
- 黑白对比
- 留白制造张力

### 5. 颜色克制

**主色调**：
- 深色（--ink）：用于标题、边框、引用框背景
- 浅色（--paper）：用于背景、引用框文字
- 灰色（--ink-muted）：用于标签、日期
- 强调色（--accent）：用于高亮框边框、关键文字

**规则**：
- 一篇文章只用一套主题色
- 颜色不超过 4 种
- 强调色只用于关键元素

## Vogue 审美关键词

用户明确要求的审美方向：

| 关键词 | 含义 | 实现方式 |
|--------|------|----------|
| 高端 | 高级感、品质感 | 衬线字体、克制用色 |
| 冷 | 冷静、理性 | 低饱和度、黑白对比 |
| 绝对控制感 | 每个元素都有意义 | 无装饰、结构清晰 |
| 版面有攻击性 | 视觉张力 | 大号数字、强烈对比 |

**不要**：
- 太像 SaaS（圆角、渐变、阴影）
- 太像说明书（居中、对称、安全）
- 太平均（无层次、无重点）
- 太安全（无张力、无个性）

## Guizang-ppt 设计原则

来自 `guizang-ppt-skill` 的核心原则：

### 1. 衬线大标题 + 非衬线正文

- 标题用衬线字体（Noto Serif SC / Playfair Display）
- 正文用非衬线字体（Noto Sans SC / Inter）
- 元数据用等宽字体（SF Mono / IBM Plex Mono）

### 2. 克制优于炫技

- WebGL 背景只在 hero 页透出
- 不用阴影、浮动卡片、padding box
- 一切信息靠字号 + 字体对比 + 网格留白

### 3. 图片是第一公民

- 图片只裁底部，保证顶部和左右完整
- 网格用 `height:Nvh` 固定，不用 `aspect-ratio`
- 图片比例必须匹配落位

### 4. 节奏靠 hero 页

- hero 和 non-hero 交替，才不累眼睛
- 每 3-4 页插入 1 个 hero 页

## 微信公众号适配

### 135 编辑器兼容性

- **全选可复制**：HTML 可直接全选复制到 135 编辑器
- **样式内联**：所有 CSS 使用 style 属性，不依赖外部样式表
- **无 JavaScript**：不使用任何 JS
- **无外部资源**：不引用外部字体、图片、CSS

### 微信公众号限制

- 不支持外部字体（需用系统字体）
- 不支持 JavaScript
- 不支持外部 CSS
- 容器宽度建议 677px

### 系统字体回退

```css
/* 衬线字体 */
font-family: 'Noto Serif SC', 'Songti SC', 'STSong', serif;

/* 非衬线字体 */
font-family: -apple-system, BlinkMacSystemFont, 'Helvetica Neue', 'PingFang SC', 'Noto Sans SC', sans-serif;

/* 等宽字体 */
font-family: 'SF Mono', 'Menlo', 'Consolas', monospace;
```
